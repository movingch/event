# 예약 확인 알림 연동 방안 v12

## 현재 반영된 방식

v12에는 네이버 클라우드 SENS를 통해 예약 완료 문자를 발송할 수 있는 Vercel 서버리스 함수가 추가되었습니다.

- 프론트엔드: `app.js`
- 발송 API: `api/send-sms.js`
- 배포 권장: Vercel
- 환경변수 예시: `.env.example`

예약이 `확정`되면 앱이 `/api/send-sms`로 예약 정보를 보내고, 서버리스 함수가 SENS API에 LMS 발송을 요청합니다. `대기` 신청은 자동 발송하지 않고, 관리자가 확정 처리 후 문자발송 버튼으로 보낼 수 있습니다.

## 발송 문구

```text
홍길동 님 예약이 완료되었습니다.
예약번호: rsv-...
영화명: 얼굴
좌석/인원: A-002 / 1명
일시: 9월 9일 (수) 오후 07:00
장소: 동천농협강당
상영 당일 현장에서 예약번호와 신청자 이름을 알려주세요.
```

## 왜 Vercel 함수가 필요한가

네이버 SENS 발송에는 Access Key, Secret Key, Service ID, 등록 발신번호가 필요합니다. 이 값을 브라우저 코드에 넣으면 누구나 볼 수 있으므로 안전하지 않습니다. 따라서 브라우저가 직접 SENS를 호출하지 않고, Vercel 서버리스 함수가 비밀키를 보관한 상태에서 대신 호출합니다.

## Vercel 환경변수

```text
NAVER_CLOUD_ACCESS_KEY=네이버클라우드 Access Key ID
NAVER_CLOUD_SECRET_KEY=네이버클라우드 Secret Key
NAVER_SENS_SERVICE_ID=ncp:sms:kr:...
NAVER_SENS_FROM=등록된 발신번호 숫자만
SMS_DRY_RUN=true
```

선택값:

```text
ALLOWED_ORIGIN=https://your-domain.vercel.app
```
처음에는 `SMS_DRY_RUN=true`로 실제 발송 없이 테스트하고, 운영 전 마지막 테스트에서 `SMS_DRY_RUN=false`로 바꾸거나 삭제합니다.


## 운영 단계별 추천

### 1단계: 지금 가능한 배포형 시제품

- Vercel에 웹앱과 `api/send-sms.js` 배포
- 예약 확정 시 문자 자동 발송
- 관리자 화면에서 문자 재발송
- 데이터는 브라우저 `localStorage`에 저장

이 단계는 기능 검증과 화면 확인에는 좋지만, 여러 관객의 신청을 운영진 한 화면에 모으기에는 부족합니다.

### 2단계: 실제 행사 운영형

- Supabase 또는 Firebase DB 연결
- 예약을 서버 DB에 저장
- 정원 초과와 지정좌석 중복을 서버에서 차단
- DB 저장 성공 후 SENS 문자 발송
- 관리자 대시보드가 DB에서 전체 통계를 조회

실제 QR을 공개 배포할 때는 2단계를 권장합니다.

### 3단계: 알림톡 확장

- 카카오 알림톡 템플릿 등록
- 예약 완료, 대기 접수, 예약 취소, 행사 전날 안내 템플릿 운영
- 알림톡 실패 시 SENS SMS로 대체 발송

## 장애 대비

- 문자 발송 실패 시 예약 자체는 유지됩니다.
- 신청 완료 팝업에서 예약정보 복사, 카톡·공유, 문자 앱 열기, 메일 열기를 사용할 수 있습니다.
- 관리자 신청자 명단에서 확인문구 복사와 문자 재발송을 사용할 수 있습니다.
- CSV 다운로드 후 외부 문자 발송 서비스로 일괄 전송할 수 있습니다.
